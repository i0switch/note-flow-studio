# 1. 要件定義書

## 1-1. システム名称
仮称: note Local Draft Studio

## 1-2. 背景
note 記事作成では以下の課題がある。

- 記事の構成作成、本文作成、整形に時間がかかる
- 記事スタイルや文体を都度再現するのが面倒
- 参考資料を読み込ませた記事作成を毎回手動で整える必要がある
- 記事内容に合ったグラフや図解を別工程で作るのが重い
- 画像生成を記事ごとに切り替えたい
- 記事内容に沿ったアイキャッチを毎回考えるのが負担
- 無料部分から有料部分への導線設計が属人的になりやすい
- note 販売設定や価格設定を毎回手動で入力するのが非効率
- note への保存を手動で行うのが非効率
- 複数アカウントや複数テンプレートを使い分けたい
- ジャンルごとに構成や訴求を切り替えたい
- note ログイン済み状態を安定的に再利用したい
- 将来、note 以外の媒体にも派生させたい

## 1-3. 目的
本アプリは、指定キーワード、参考資料、販売設定から note 用記事を生成し、必要に応じて画像・グラフ生成を行い、note に下書き保存し、販売導線の下書きまで整える作業を効率化することを目的とする。

## 1-4. 対象ユーザー
- 主対象: 購入者本人
- 想定利用者: 1人利用中心
- 将来想定: 少人数運用、媒体追加、機能追加

## 1-5. 解決したい業務課題
- note 記事の量産効率向上
- 参考資料ベースの記事品質向上
- 記事スタイルの再現性向上
- オールジャンル運用時のテンプレート切替最適化
- グラフ、画像、アイキャッチ制作の一体化
- 無料部分から有料部分への導線品質向上
- 複数生成AIの使い分け
- note アカウントごとの運用最適化
- 下書き保存までの作業自動化
- note 販売設定の準自動化
- ブラウザログイン状態の使い回し最適化

## 1-6. スコープ

### スコープ内
- 記事生成
- 参考資料取り込み・管理
- プロンプトテンプレート管理
- ジャンル別生成制御
- note アカウント管理
- AIプロバイダ設定管理
- 画像生成設定管理
- グラフ生成
- アイキャッチ生成
- 無料部分 / 有料部分の自動分割支援
- 販売導線文生成
- note 販売設定自動入力
- note 下書き保存
- フォールバック実行
- PinchTab 連携
- Playwright 連携
- 実行履歴管理
- 再編集・再生成
- 将来の X 出力基盤のための内部構造保持

### スコープ外
- note の公開実行
- 公開スケジュール実行
- チーム向け詳細権限制御
- クラウド同期
- note 売上分析
- 外部決済基盤の課金管理

## 1-7. 業務要件
1. ユーザーはキーワード等を指定して記事生成を開始できること
2. ユーザーは保存済みプロンプト名を選択できること
3. ユーザーは note アカウントを選択できること
4. ユーザーは画像生成有無を選択できること
5. ユーザーは補足指示を入力できること
6. ユーザーは URL、テキスト、ファイルを参考資料として取り込めること
7. ユーザーは対象ジャンルを手動指定または自動推定で選べること
8. システムは選択したAIプロバイダを用いて参考資料付き記事を生成できること
9. システムは画像生成が必要な場合に画像生成ジョブを実行できること
10. システムは記事内容に沿ったアイキャッチ画像を生成できること
11. システムは記事内容に沿ったグラフを生成できること
12. システムは無料部分と有料部分の境界、および誘導文を生成できること
13. システムは note 販売向けの価格、導線文、販売設定候補を保持できること
14. システムは note 非公式APIで下書き保存を試みること
15. 非公式API失敗時、条件に応じて Playwright または PinchTab にフォールバックできること
16. システムは必要に応じて note 販売設定をブラウザ自動操作で入力できること
17. 実行結果を履歴として保持できること
18. 生成済み記事を再編集・再保存できること
19. note アカウントごとにブラウザ自動操作方式を分けられること
20. PinchTab のプロファイルを note アカウントごとに管理できること

## 1-8. 非機能要件

### 性能
- 一般的な1記事生成は数十秒〜数分以内を目安とする
- UI操作に対し、主要画面の初期表示はローカル環境で快適に行えること

### 可用性
- ローカル動作のため、アプリ自体は単体PCで完結可能であること
- AI API未接続時でも設定閲覧や履歴閲覧は可能であること
- PinchTab 未使用時でも Playwright で運用可能であること

### 保守性
- AIプロバイダ追加が容易な構成であること
- 媒体アダプタ追加が容易な構成であること
- ブラウザ操作アダプタを差し替え可能な構成であること
- ジャンル追加や販売導線テンプレート追加が容易な構成であること

### セキュリティ
- APIキーは安全にローカル保存すること
- 認証情報の表示はマスクすること
- note セッション情報は分離管理すること
- PinchTab トークンを安全に保持すること
- 参考資料内の秘匿情報を不要にログ出力しないこと

### 拡張性
- 将来、X用派生出力に対応できること
- 将来、媒体アダプタ追加に対応できること
- 将来、ブラウザ操作先を note 以外へ広げられること
- 将来、販売導線を note 以外の媒体へ転用できること

## 1-9. 成功条件
- 指定キーワードから再現性のある記事生成ができる
- 参考資料を踏まえた記事、グラフ、アイキャッチを一体生成できる
- 無料部分から有料部分への導線を含む販売用記事が作れる
- note 下書き保存まで準自動化できる
- note 販売設定入力まで準自動化できる
- 複数AI / 複数noteアカウント / 複数プロンプトの運用が成立する
- オールジャンル運用を1つの内部基盤で扱える
- Playwright と PinchTab の両方を選択可能にできる
