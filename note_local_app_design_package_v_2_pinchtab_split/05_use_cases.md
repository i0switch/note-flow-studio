## 5-2. 主要ユースケース記述

### UC-001 記事を新規生成する
**事前条件**
- AI設定が完了している
- 使用するアカウントが登録済みである
- 使用するプロンプトが登録済みである

**基本フロー**
1. ユーザーは記事生成画面を開く
2. キーワードを入力する
3. 対象ジャンルを選択または自動推定する
4. 使用アカウントを選択する
5. プロンプト名を選択する
6. 必要に応じて参考資料を選択する
7. 画像有無とグラフ有無を選択する
8. 販売モードを選択する
9. 補足指示を入力する
10. 実行ボタンを押す
11. システムは記事生成を実行する
12. システムは結果を保存する
13. システムは結果を表示する

### UC-003 noteへ下書き保存する
**基本フロー**
1. システムは note 非公式APIで保存を試行する
2. 販売設定対象の記事であれば販売設定の適用可否を判定する
3. 保存成功時、下書きURLを取得する
4. 実行履歴に保存する

### UC-004 note保存失敗時にフォールバックする
**基本フロー**
1. 非公式API保存が失敗する
2. システムは再試行条件を判定する
3. 条件一致時、設定されたブラウザ操作方式へ切り替える
4. Playwright または PinchTab で保存を試行する
5. 成功時、下書きURLを履歴に保存する

### UC-015 PinchTab を設定する
**基本フロー**
1. ユーザーはブラウザ操作設定画面を開く
2. PinchTab の Base URL を入力する
3. 必要に応じてトークンを入力する
4. 接続確認を行う
5. 成功時、システムは設定を保存する

### UC-017 参考資料を取り込む
**基本フロー**
1. ユーザーは参考資料管理画面を開く
2. URL、テキスト、またはファイルを指定する
3. システムは内容を抽出して保存する
4. ユーザーはタグとジャンルを付与する
5. ユーザーは記事生成ジョブへ紐付ける

### UC-018 グラフ付き記事を生成する
**基本フロー**
1. ユーザーは記事生成時にグラフ生成を有効にする
2. システムは参考資料または本文からグラフ要素を抽出する
3. システムはグラフ定義を生成する
4. システムはグラフ画像または埋め込み用データを保存する
5. 記事詳細画面でプレビューする

### UC-019 無料部分から有料部分への導線を生成する
**基本フロー**
1. ユーザーは販売モードを無料+有料に設定する
2. システムは無料部分と有料部分の境界を提案する
3. システムは無料部分末尾の誘導文を生成する
4. ユーザーは必要に応じて内容を編集する
5. システムは販売用記事として保存する

### UC-020 note販売設定を自動入力する
**基本フロー**
1. ユーザーは価格と無料公開範囲を確定する
2. システムは利用可能な保存方式を判定する
3. 非公式APIで不足する設定はブラウザ自動操作で補完する
4. システムは価格、無料公開範囲、販売状態を入力する
5. システムは下書きURLと設定結果を保存する
