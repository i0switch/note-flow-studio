# 8. 業務ルール・例外ルール集

## 8-1. 業務ルール
1. 記事生成と note 保存は別工程として扱う
2. 画像生成は本文生成と別工程として扱う
3. note 保存に失敗しても生成済み記事は削除しない
4. 実行履歴は成功・失敗に関わらず残す
5. note アカウントごとに別設定を持つ
6. 品質ルールはアプリ固定ロジックではなくプロンプトで制御する
7. 公開処理は行わず、下書き保存までを対象とする
8. 生成記事は内部構造化形式と媒体向け整形形式の両方を保持する
9. 画像生成失敗時でも本文保存は続行する
10. 再編集後の再保存を許可する
11. ブラウザ操作方式はアカウントごとに設定可能とする
12. 参考資料の原文と要約を保持し、生成根拠を追跡可能にする
13. オールジャンル対応はジャンル別テンプレート切替で実現する
14. 販売用記事では無料部分と有料部分の境界を保持する

## 8-2. note保存ルール
1. 既定は非公式API優先とする
2. 非公式APIでタイムアウト、5xx、一時的通信失敗が発生した場合は再試行またはフォールバック判定を行う
3. 認証失敗時は即エラーとし、自動フォールバックしない
4. ブラウザ操作フォールバック先はアカウント設定に従う
5. Auto 設定時は Playwright を優先し、PinchTab は第2候補とする
6. 販売設定が必要な記事で非公式APIが対応できない場合、ブラウザ自動操作で補完する

## 8-3. フォールバックルール
1. 非公式API保存失敗時、再試行回数以内であれば再試行する
2. 再試行上限到達後、設定されたブラウザ操作方式へ移行する
3. Playwright 失敗時、PinchTab 有効なら PinchTab を試行可能とする
4. PinchTab 失敗時、Playwright 有効なら Playwright を試行可能とする
5. すべての試行結果を save_attempts に記録する

## 8-4. PinchTab運用ルール
1. PinchTab はローカル接続を前提とする
2. PinchTab トークンを使用する場合、秘匿情報として扱う
3. PinchTab プロファイルは note アカウントごとに分離する
4. PinchTab 未起動時は接続エラーとして扱う
5. PinchTab 連携は無効化可能とする

## 8-5. 参考資料ルール
1. 参考資料は URL、テキスト、ファイルのいずれかで登録可能とする
2. 抽出失敗した資料は下書き保存前に警告表示する
3. 参考資料の要約は編集可能とする
4. 参考資料の原文はログへ全文出力しない

## 8-6. 画像・アイキャッチ生成ルール
1. 画像生成の有無はジョブ単位で選択可能とする
2. 画像生成方式はプロファイルで管理する
3. 画像生成失敗時は本文処理を止めない
4. アイキャッチと本文挿絵は役割を分けて保持する
5. アイキャッチは記事タイトル、要点、ジャンルを踏まえて生成する

## 8-7. グラフ生成ルール
1. グラフ生成は参考資料または本文内の構造化情報を根拠に行う
2. 根拠が曖昧な場合はグラフ生成をスキップし、理由を表示する
3. グラフ定義と元データの対応関係を保存する

## 8-8. 無料→有料導線ルール
1. 無料部分だけで最低限の価値が伝わる構成にする
2. 誘導文は煽りだけで終わらず、有料部分で得られる具体価値を示す
3. 無料部分と有料部分の境界はユーザーが上書き可能とする
4. 販売モードが通常の記事では誘導文を自動挿入しない

## 8-9. note販売自動化ルール
1. 価格、無料公開範囲、誘導文は販売プロファイルまたはジョブ指定値を優先する
2. note 側仕様で自動入力できない項目がある場合はユーザーへ明示する
3. 販売設定失敗時でも記事本文と画像は保持する
4. 自動販売設定は公開実行を含まない

## 8-10. 例外ルール

### AI API接続不可
- 記事生成は失敗
- UIへ接続エラー表示
- 実行履歴へ失敗記録保存

### noteセッション期限切れ
- 該当ブラウザ操作方式での保存は失敗
- 接続再確認を促す
- セッション状態を expired に更新

### PinchTab疎通不可
- PinchTab 保存は失敗
- 代替方式があればそちらへ移行
- ログへ接続失敗記録を残す

### 画像生成プロバイダ異常
- 画像生成のみ失敗扱い
- 本文生成結果は保持

### 参考資料解析失敗
- 該当資料を無効扱いにする
- 記事生成は他資料のみで継続可能とする
- エラー理由を記録する

### グラフ生成不可
- グラフのみ失敗扱い
- 記事本文とアイキャッチは保持する

### 販売設定反映失敗
- note販売設定のみ失敗扱い
- 下書き本文保存に成功していれば記事は保持する
- 再試行と手動編集導線を案内する
