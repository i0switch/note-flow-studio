# DEVELOPMENT_ROADMAP.md

## 方針
開発は「基盤 -> 最短価値 -> 拡張機能」の順で進める。
記事生成アプリとしてまず成立させ、その後に販売導線とブラウザ自動化の厚みを増やす。

## Phase 0: 土台決め
- 技術スタック決定
- ディレクトリ構成決定
- コーディング規約決定
- SQLite とマイグレーション方針決定
- `.env` と秘密情報の扱い決定

## Phase 1: MVP
- 記事生成画面
- 参考資料管理
- プロンプト管理
- noteアカウント管理
- AI設定
- オールジャンル生成制御
- 無料→有料導線生成
- 記事連動アイキャッチ生成
- SQLite 保存
- 実行履歴
- note 非公式API 保存
- Playwright 保存アダプタ

## Phase 2: 実運用強化
- PinchTab 設定画面
- PinchTab 保存アダプタ
- 再編集機能
- 画像生成機能の強化
- グラフ生成機能
- note 販売設定自動化
- 接続診断

## Phase 3: 拡張
- X 派生出力
- エクスポート機能
- ライセンス認証
- 更新機能

## 各Phaseの完了条件

### Phase 0
- 技術スタックが文書化されている
- 初期ディレクトリが作成されている
- 開発開始手順が README に反映されている

### Phase 1
- 記事生成から下書き保存まで一連で動く
- 参考資料、ジャンル、販売モードを保持できる
- 保存失敗時に Playwright フォールバックが動く

### Phase 2
- PinchTab を含めた保存経路が動く
- 販売設定が自動反映できる
- グラフや画像の失敗が全体停止に繋がらない

### Phase 3
- note 以外の派生出力が可能
- 配布・更新の運用まで視野に入る

## Codexへの期待アクション
- 毎回「今どのPhaseか」を明示して進める
- 新規実装時は、Phase外の拡張を混ぜすぎない
- 仕様不足を見つけたら、即実装せず論点として残す
