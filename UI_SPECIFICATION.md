# UI_SPECIFICATION.md

## 目的

この文書は、`note Local Draft Studio` の現行 Web UI を改善するための実装基準書。
既存の設計書 `note_local_app_design_package_v_2_pinchtab_split/02_screen_specifications.md` を補完しつつ、
現在の実装に合わせて画面構成、導線、状態、改善論点を整理する。

対象は `localhost` で動く購入者向けローカルアプリ。

## UI の前提

- 利用環境: Windows ローカルアプリ
- 配布形態: `portable`
- 主利用フロー:
  1. 初回セットアップ
  2. 記事生成
  3. 記事詳細確認
  4. note 下書き保存 / 公開
  5. 参考資料管理
  6. 設定 / 診断

## 情報設計

### グローバルレイアウト

- 左サイドバー固定ナビゲーション
- 右側メインコンテンツ

## 画面仕様

### SCR-100 初回セットアップ画面

**目的**
- 購入者が `.env` を直接編集せず初回設定できるようにする
- 起動前に依存関係の状態を確認できるようにする

**主な表示ブロック**
- ヒーロー説明
- Step 1: 初回設定フォーム
- Step 2: 依存チェック一覧
- Step 3: 配布前チェック

**入力項目**
- Gemini API Key
- Gemini Model
- note ID
- note Password
- PinchTab URL
- PinchTab Profile
- PinchTab Port
- localhost Port
- Playwright ヘッドレス実行トグル

**操作**
- 設定を保存
- Chromium を導入

**表示状態**
- `未設定`
- `設定済み`
- 依存チェック `ok / warn / error`
- 保存中 / 導入中

**バリデーション方針**
- ポートは数値
- 必須項目不足時は API 側エラーをメッセージ表示
- `note Password` はマスク表示

**改善観点**
- APIキー入力の安全性表示
- どの項目が必須かの視覚区別
- `warn` の対処法を行内に出す
- 保存成功後の次アクション導線を強化する

### SCR-001 記事生成画面

**目的**
- 記事生成ジョブの条件入力
- 記事生成開始の起点

**主な入力項目**
- キーワード
- 対象ジャンル
- 使用アカウント
- プロンプト
- 想定価格
- 販売モード
- 参考資料
- 補足指示
- 画像トグル
- グラフトグル
- 販売導線トグル

**主な操作**
- 生成開始

**依存データ**
- note アカウント一覧
- プロンプトテンプレート一覧
- 参考資料一覧

**改善観点**
- 重要項目の優先度整理
- 参考資料選択UIの検索性改善
- 生成条件のプリセット保存
- 生成開始前の確認プレビュー

### SCR-002 実行履歴一覧

**目的**
- 過去ジョブ一覧の把握
- 詳細画面への導線

**表示項目**
- キーワード
- ジャンル
- 販売モード
- 状態
- 詳細リンク

**改善観点**
- 実行日時がまだ見えない
- 保存結果や公開結果の列が足りない
- ステータス色分けを追加したい
- フィルタ、検索、並び替えが必要

### SCR-003 記事詳細画面

**目的**
- 生成済み記事の内容確認
- 保存、公開、販売設定反映の実行

**主な表示項目**
- タイトル
- 無料部分
- 有料導線
- 有料部分
- 本文
- アイキャッチ / グラフ要約
- 参考資料一覧
- 保存履歴
- 実行ログ

**主な操作**
- グラフ再生成
- 販売設定反映
- noteへ下書き保存
- noteへ公開

**改善観点**
- 保存ボタン群の優先順位整理
- 成功 / 失敗結果の視覚区別
- アセットの実画像プレビュー不足
- 本文が長い時の可読性改善
- 保存履歴に時系列や再試行理由を追加したい

### SCR-013 参考資料管理画面

**目的**
- 参考資料の追加と再利用

**主な入力項目**
- タイトル
- ソース種類
- ジャンル
- タグ
- 本文 / URL / ファイルパス

**主な操作**
- 取り込む

**主な表示項目**
- 登録済み参考資料一覧
- タイトル
- 要約

**改善観点**
- 一覧に検索、タグ絞り込みがない
- 取込結果の成功 / 失敗表示が弱い
- ファイル取込時の補助説明が足りない
- ジョブ紐付け状況も見せたい

### SCR-010 / SCR-011 設定 / 診断画面

**目的**
- 基本設定の変更
- 現在の診断結果確認
- note アカウント追加

**主な入力項目**
- localhost Port
- Gemini Model
- note アカウント表示名
- 保存優先順位

**主な表示項目**
- 診断一覧
- セットアップ依存一覧
- note アカウント一覧

**主な操作**
- 設定を保存
- アカウント追加

**改善観点**
- 設定と診断が1画面に混在していて重い
- 診断項目ごとの再実行ボタンがない
- account 設定項目が最小限すぎる
- Playwright / PinchTab 個別設定の編集UIが不足

## 共通コンポーネント仕様

### Sidebar
- グローバルナビゲーション
- 役割は移動のみ
- 現在地ハイライトは未実装

### Field
- ラベル + 入力欄
- 画面全体で統一利用

### Toggle
- Boolean 入力
- `画像 / グラフ / 販売導線 / Playwrightヘッドレス` に利用

### InfoCard
- 長文表示カード
- 記事詳細画面で使用

### LoadingCard
- 遅延読込中と初期待機の共通UI

## 状態設計

### 初期起動

| 条件 | 表示 |
| --- | --- |
| `setup-status.isConfigured = false` | セットアップ画面 |
| `setup-status.isConfigured = true` | 通常アプリ |

### 通常アプリ起動後

| 条件 | 表示 |
| --- | --- |
| 依存チェックに `warn/error` がある | 上部警告バナーを表示 |
| 依存チェックがすべて `ok` | 警告バナー非表示 |

### 非同期状態

- `isLoading`
- `mutation.isPending`
- `success`
- `error`
- `refetchInterval` による定期再取得

## UI改善の優先順位

### 優先度A
- ダッシュボードの情報密度整理
- 実行履歴の検索 / フィルタ / ソート
- 記事詳細の保存結果視認性向上
- 設定画面の分割

### 優先度B
- サイドバーの現在地表示
- 参考資料管理の検索、タグフィルタ
- セットアップ画面のエラー説明強化
- note 公開結果の視覚フィードバック改善

### 優先度C
- キーボード操作最適化
- トースト通知
- ダークテーマ検討
- 最近使った条件の復元

## 改善方針

### 1. 画面の責務を整理する
- ダッシュボードは「生成開始」と「進行中ジョブ確認」に寄せる
- 履歴は独立ページ化を検討する
- 設定と診断は分離を検討する

### 2. 成功 / 失敗 / 警告を色と構造で明確化する
- ログ文字列頼みを減らす
- ステータスバッジを増やす
- 保存経路の結果をカード化する

### 3. note 運用導線を強くする
- 下書き保存と公開を目的別に分ける
- `free_paid` のときだけ販売設定ブロックを強調表示する
- 公開前確認モーダルを入れる

### 4. 購入者向けの迷いを減らす
- 初回セットアップの説明文をさらに具体化する
- 設定未完了箇所を上部に集約する
- `いま次に何をすればいいか` を常に出す

## 画面改善時の受け入れ条件

- 初回利用者がセットアップから記事生成まで迷わず進める
- 保存 / 公開 / 販売設定反映の結果がひと目でわかる
- 長文記事でも詳細画面が読みやすい
- 参考資料と設定画面が情報過多で崩れない
- モバイル幅でも最低限操作できる
